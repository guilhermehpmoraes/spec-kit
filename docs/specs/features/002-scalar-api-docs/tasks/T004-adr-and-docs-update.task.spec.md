### 4.5 ADR-010 Structure

File: `docs/decisions/010-scalar-api-documentation.md`

**Title**: Scalar as Standard API Documentation UI

**Status**: Accepted

**Context**:
- The monorepo uses `@nestjs/swagger` to generate OpenAPI specs for all NestJS backends.
- The default Swagger UI (`SwaggerModule.setup`) is functional but dated — limited navigation, no persistent auth, basic visual design.
- Scalar is an open-source API reference UI that renders OpenAPI specs with modern design, built-in request testing, and persistent authentication.
- `@scalar/nestjs-api-reference` integrates via Express middleware, consuming the same OpenAPI document generated by `SwaggerModule.createDocument`.

**Decision**:
1. Replace `SwaggerModule.setup()` with `@scalar/nestjs-api-reference` (`apiReference` middleware) in all NestJS backends.
2. Configure Scalar with Bearer Auth pre-selected (`preferredSecurityScheme: "bearer"`) and persistent authentication (`persistAuth: true`).
3. Include a step-by-step authentication guide in `DocumentBuilder.setDescription()`.
4. Establish a decorator standard: every `@ApiProperty` must include `description` and `example`; every non-void endpoint must have a typed response DTO in `@ApiResponse({ type: ... })`.
5. `@nestjs/swagger` remains the OpenAPI generation engine — only the UI layer changes.

**Consequences**:
- Positive: Better DX (persistent auth, modern UI, built-in request testing), consistent documentation quality via decorator standard.
- Positive: No API contract changes — drop-in UI replacement.
- Negative: Adds `@scalar/nestjs-api-reference` as a new dependency.
- Neutral: All future backends must follow the Scalar setup pattern and decorator standard.

```
│   │   ├── dto/
│   │   │   ├── login.dto.ts         # LoginDto (email, senha)
│   │   │   ├── refresh-token.dto.ts # RefreshTokenDto (refreshToken)
│   │   │   └── logout.dto.ts        # LogoutDto (refreshToken)
```
→
```
│   │   ├── dto/
│   │   │   ├── login.dto.ts             # LoginDto (email, senha)
│   │   │   ├── login-response.dto.ts    # LoginResponseDto (accessToken, refreshToken)
│   │   │   ├── refresh-token.dto.ts     # RefreshTokenDto (refreshToken)
│   │   │   ├── refresh-response.dto.ts  # RefreshResponseDto (accessToken)
│   │   │   └── logout.dto.ts            # LogoutDto (refreshToken)
```

**Change 3** — Add a new section "API Documentation Standard" after the "Shared Packages" table (before "References"):

Content should describe:
- Scalar is served at `/api/docs` via `@scalar/nestjs-api-reference`
- `@nestjs/swagger` generates the OpenAPI spec; Scalar renders it
- Bearer Auth persistence is configured via `persistAuth: true`
- Every `@ApiProperty` must include `description` and `example`
- Every non-void endpoint must have a typed `@ApiResponse({ type: ResponseDto })`
- Reference ADR-010
